# 自动登录功能实现计划

## 概述

为日报数据获取插件添加自动登录功能，支持用户名/密码+图片验证码识别自动完成登录。

## 现状分析

- 现有插件通过 Cookie 中的 `Admin-Token` 验证用户身份
- 登录页面在 `https://gs.capinfo.com.cn` 域名下
- 现有功能依赖用户手动登录后再获取数据

## 实现方案

### 1. 新增登录模块 (content.js)

在 `content.js` 中添加自动登录逻辑：

- `checkLoginPage()`: 检测是否在登录页面
- `fetchCaptchaImage()`: 获取验证码图片
- `solveCaptcha(imageUrl)`: 调用 Gitee AI API 识别验证码
- `submitLogin(username, password, captcha)`: 提交登录表单
- `autoLogin()`: 主编排函数，协调各步骤

### 2. 登录状态管理 (background.js)

扩展 `background.js` 支持登录相关消息：

- 接收来自 popup 的登录请求
- 转发登录状态查询

### 3. 登录 UI (popup.html / popup.js)

- 添加"自动登录"按钮
- 添加登录状态显示
- 处理登录结果反馈

### 4. 验证码识别服务 (popup.js)

新增 `solveCaptcha(imageUrl)` 函数：

- 调用 Gitee AI API (`https://ai.gitee.com/v1/chat/completions`)
- 使用模型 `Qwen2.5-VL-7B-Instruct`
- System prompt: "仅输出计算结果"
- 返回识别结果

### 5. 配置文件

新增 `config.js` 存放登录凭证：

```javascript
const LOGIN_CONFIG = {
  username: '13301091227',
  password: 'lrjian2290'
};
```

## 文件修改清单

| `popup.html`    | 添加登录按钮和状态显示区域                    |
| `popup.js`      | 添加登录触发逻辑和验证码识别调用                 |

| `manifest.json` | 无需修改（已有必要权限）                     |

## 验证码识别流程

1. 登录页面获取验证码图片 URL
2. 将图片 URL 发送给 Gitee AI API
3. 构造 messages 格式：
   - system: "仅输出计算结果"
   - user: 包含 image\_url 和 "Please describe this image"
4. 解析 API 返回的 content 即为验证码结果

## 登录页面检测逻辑

检测元素：

- 用户名输入框 (可能的选择器: `#username`, `[name="username"]`, etc.)
- 密码输入框
- 验证码图片
- 登录按钮

## 验证步骤

1. 点击"自动登录"按钮
2. 插件自动识别验证码
3. 自动填写表单并提交
4. 显示登录结果（成功/失败）
5. 登录成功后可正常获取日报数据
